# Git Branch Naming Rules

## Purpose
Git branch naming conventions for features, fixes, releases, and hotfixes

## Rules

### Rule 1: Consistency
All team members must follow these standards consistently across the codebase.

### Rule 2: Review Process
Changes to these rules require team discussion and approval.

### Rule 3: Enforcement
Automated checks should enforce these rules where possible via CI/CD pipelines.

## Exceptions
Exceptions must be documented with a clear justification and approved by the team lead.
